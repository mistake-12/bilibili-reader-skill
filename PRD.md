# B站视频阅读推送 Skill — 产品需求文档

## 1. 产品概述

### 1.1 产品名称
`bilibili-reader` — B站收藏夹视频智能总结Skill

### 1.2 产品愿景
解决"收藏了但不看"的痛点。用户在B站收藏了大量视频却很少回看，本skill通过定期随机选取收藏视频，自动生成结构化的中英双语总结PDF，帮助用户高效利用收藏内容。

### 1.3 目标用户
- B站重度用户，收藏夹中有大量未看视频
- 希望系统性消化收藏内容的用户
- hermes agent用户

## 2. 核心功能

### 2.1 功能流程
```
用户触发 → 获取收藏夹列表 → 用户选择收藏夹（多选时询问）
         → 随机选取未处理视频 → 获取视频数据（字幕/评论/弹幕）
         → LLM生成中英双语总结 → 生成PDF文档
         → 记录已处理视频 → 输出PDF路径
```

### 2.2 功能模块

| 模块 | 功能 | 优先级 |
|------|------|--------|
| B站数据获取 | 获取收藏夹、视频信息、字幕、评论、弹幕 | P0 |
| LLM总结 | 生成中英双语结构化总结 | P0 |
| PDF生成 | 排版美观的PDF文档 | P0 |
| 长期记忆 | 记录已处理视频，避免重复 | P0 |
| 收藏夹选择 | 多收藏夹时询问用户选择 | P1 |
| 微信推送 | 通过微信发送PDF（后续扩展） | P2 |

## 3. 技术架构

### 3.1 技术栈
- **语言**：Python 3.11+
- **HTTP客户端**：requests
- **弹幕解析**：protobuf
- **PDF生成**：Jinja2 + weasyprint
- **配置管理**：python-dotenv
- **Skill框架**：hermes agent SKILL.md格式

### 3.2 外部依赖
- B站API（需登录态Cookie）
- LLM API（通过hermes agent或直接调用）
- weasyprint系统依赖（GTK/Cairo等）

### 3.3 数据流
```
B站API → 原始数据（JSON/protobuf）
       → 数据清洗和提取
       → LLM总结 prompt
       → 结构化总结（JSON）
       → HTML模板渲染
       → PDF文件输出
```

## 4. API设计

### 4.1 B站API端点

| 端点 | 用途 | 认证 |
|------|------|------|
| `/x/v3/fav/folder/created/list-all` | 获取收藏夹列表 | Cookie |
| `/x/v3/fav/resource/list` | 获取收藏夹内容 | Cookie |
| `/x/web-interface/view` | 获取视频详情 | Cookie |
| 字幕URL（从视频详情获取） | 下载字幕 | 无需 |
| `/x/v2/reply/main` | 获取评论 | Cookie |
| `/x/v2/dm/web/seg.so` | 获取弹幕 | Cookie |

### 4.2 Cookie配置
需要以下Cookie字段：
- `SESSDATA`：会话数据
- `bili_jct`：CSRF令牌
- `buvid3`：设备标识

```json
    }
  ],
  "stats": {
    "total_processed": 42,
    "last_processed_at": "2026-05-04T10:00:00"
  }
}
```

## 7. 配置项

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `BILIBILI_SESSDATA` | B站Cookie SESSDATA | 必填 |
| `BILIBILI_BILI_JCT` | B站Cookie bili_jct | 必填 |
| `BILIBILI_BUVID3` | B站Cookie buvid3 | 必填 |
| `OUTPUT_DIR` | PDF输出目录 | `./output` |
| `MAX_COMMENTS` | 高赞评论数量 | 10 |
| `MAX_DANMAKUS` | 弹幕数量 | 50 |
