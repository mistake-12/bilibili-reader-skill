# 视频文案：收藏了不看？让 AI 帮你做视频笔记

> 目标平台：B站
> 预计时长：3-4 分钟
> 优化目标：完播率 · 点赞率 · 转发率 · 评论率

---

## 开场钩子（0:00 - 0:15）

**[画面：B站收藏夹页面，快速滚动，数字飞速跳动]**

> 你的B站收藏夹里，有多少个视频？
>
> 我的是 347 个。
>
> 你真正看完的，有几个？

**[停顿 1 秒]**

> 我猜，不超过 20%。

**【设计意图】**：开头直接戳痛点。数字越具体越有冲击力。"347"比"几百个"更真实。停顿制造悬念。

---

## 痛点放大（0:15 - 0:45）

**[画面：展示收藏夹分类截图，每个分类里大量未看视频]**

> 每个B站用户都有这个病——我管它叫「收藏焦虑」。
>
> 看到好视频，收藏。"以后再看"。然后，再也没有以后。
>
> 收藏夹从 50 个涨到 500 个，焦虑感越来越强。
>
> 更扎心的是——收藏了 ≠ 学会了。你只是收藏了。

**[画面：展示一个 23 分钟的技术教程视频]**

> 23 分钟的视频，你真的有时间从头看完吗？

**【设计意图】**：共情 + 自嘲。"收藏焦虑"是造词，容易被记住和传播。"收藏了 ≠ 学会了"是金句，适合截图转发。

---

## 转折（0:45 - 1:00）

**[画面：黑屏，打字机效果]**

> 如果有一个 AI，帮你把每个视频都读一遍，
>
> 然后给你一份学习笔记呢？

**[画面：切到 bilibili-reader logo + 名称]**

> 我做了一个工具，叫 bilibili-reader。

**【设计意图】**：从"痛"到"解"的转折。问句引发好奇，答案揭晓时注意力最高。

---

## 演示过程（1:00 - 2:00）

**[画面：终端窗口，实时运行 bilibili-reader]**

> 用法很简单。对 Agent 说一句：
>
> "帮我从收藏夹随机总结一个视频"

**[画面：终端开始运行，逐行输出]**

> 它会自动：
>
> 第一步，获取你的收藏夹列表
>
> 第二步，随机选一个你没看过的视频
>
> 第三步，下载字幕、抓取高赞评论、解析弹幕
>
> 第四步，判断这个视频是什么类型——技术教程？学科教育？语言学习？
>
> 第五步，用对应的提示词，生成结构化总结

**[画面：PDF 生成成功提示]**

> 一份中英双语的 PDF 学习笔记，就这样生成了。

**【设计意图】**：实操演示是最有说服力的。逐步讲解让用户觉得"我也可以"。

---

## 效果展示（2:00 - 2:40）

**[画面：打开生成的 PDF，逐页展示]**

> 看看这份笔记里有什么：
>
> 一句话 TLDR——30 秒就知道这个视频讲了什么
>
> 核心要点 5 条——比你自己看视频记的还全
>
> 前置知识——告诉你学这个需要什么基础
>
> 难度评级——入门还是进阶，一眼就知道
>
> 常见误解——别人踩过的坑，你可以绕开
>
> 后续行动——看完之后应该做什么

**[画面：展示技术教程类的代码片段提取]**

> 如果是技术教程，它还会提取代码片段和工具清单。
>
> 如果是书籍拆解，它会提取关键引述。
>
> 10 种视频类型，10 套不同的提取策略。

**【设计意图】**：展示具体功能点，让用户感受到"这不只是一个摘要工具"。

---

## 考古系统（2:40 - 3:00）

**[画面：终端展示考古进度]**

> 更好玩的是这个——收藏夹考古系统。
>
> 每消化一个视频，进度条就涨一点。
>
> 消化 10 个，解锁「考古学徒」成就。
>
> 覆盖 5 种体裁，解锁「博览群书」成就。
>
> 你会发现自己开始主动想去"考古"了。

**【设计意图】**：游戏化是留存的关键。"成就"系统让人想继续用。

---

## 安装（3:00 - 3:30）

**[画面：终端展示安装命令]**

> 安装也超简单。
>
> Hermes Agent 用户，直接对 Agent 说：
>
> "帮我去 GitHub 安装 bilibili-reader skill"
>
> OpenClaw 用户，一行命令：
>
> openclaw skills install mistake-12/bilibili-reader-skill
>
> 然后说一句"帮我从收藏夹总结一个视频"就行了。
>
> 它会自动引导你扫码登录 B站，不需要手动复制 Cookie。

**【设计意图】**：降低安装门槛。"一句话安装"比"git clone + pip install + 配置 .env"有吸引力得多。
