> 基于 PM-Skills 框架的产品诊断与市场定位分析
> 分析日期：2026-05-05

## 一、JTBD 分析：用户的核心待完成任务

### 用户的 Job

**When** 我在B站收藏了大量视频却堆积不看时，
**I want to** 用最低的认知成本消化收藏内容，
**So I can** 不浪费当初收藏时的学习意图，把"收藏焦虑"转化为"已掌握"。

### 三层 Job 拆解

| 层级 | Job 描述 | 当前满足程度 |
|------|----------|-------------|
| **Functional** | 从收藏夹中自动选一个视频，生成可阅读的内容摘要 | ✅ 已实现 |
| **Emotional** | 消除"收藏了不看"的内疚感，获得"我在进步"的掌控感 | ⚠️ 部分满足 |
| **Social** | 能把学到的东西分享给别人，显得有见识 | ❌ 未覆盖 |

### 需求 Gap 分析

#### Gap 1：Job Driver 被低估

用户触发 skill 的场景不只是"想看视频"，更常见的是：
- 睡前碎片时间，想看点有营养的但不想看完整视频
- 通勤路上，想快速回顾之前收藏的干货
- 写文章/做分享时，想引用之前收藏的某个观点

**建议**：补充按关键词搜索和按体裁筛选的能力，从"被动随机"到"主动查找"。

#### Gap 2：输出形态不匹配使用场景

PDF 在手机上阅读体验差、无法被搜索、无法被引用、生成依赖 weasyprint。

**建议**：增加 Markdown 输出作为轻量替代。PDF 适合归档，Markdown 适合即时阅读和分享。

#### Gap 3：记忆系统只记了"处理过"，没记"学到了什么"

用户无法回答"我上周总结了哪些技术教程？"或"之前看过一个关于 Python 的视频，要点是什么？"

**建议**：记忆系统应存储 summary 的结构化数据，支持历史回顾和搜索。

#### Gap 4：Job Driver 被低估 — 用户真的会回来用第二次吗？

当前流程需要用户手动运行命令、等 API 调用、等 LLM 生成、等 PDF 渲染——整个流程 3-5 分钟。PDF 输出在本地，手机上看不到，无法分享。第一次新鲜感过后，"收藏了不看"的问题没有被解决，只是变成了"生成了 PDF 也不看"。

**核心风险：这是一个"用了觉得酷，但不会形成习惯"的工具。**

## 二、POL Probe：残酷真相

### 探针 #1：用户留存

**假设：** 用户收藏了一个视频，运行 skill 生成 PDF，会再次使用

**残酷真相：**
- 整个流程 3-5 分钟，摩擦成本高
- PDF 输出在本地 `output/` 目录，手机上看不到，无法分享
- 第一次新鲜感过后，"收藏了不看"的问题没有被解决，只是变成了"生成了 PDF 也不看"

**验证方式：** 运行 3 次后，检查 `data/processed.json` 是否有 3 条记录。

---

### 探针 #2：没有字幕的视频

**假设：** 大多数视频都有字幕

**残酷真相：**
- B站大量视频没有 CC 字幕（尤其是早期视频、游戏区、音乐区）
- 没有字幕时，只能基于标题+简介+评论生成总结——质量断崖式下降
- 用户看到一个质量很差的总结后，对整个工具的信任度会暴跌

**验证方式：** 随机取 10 个收藏视频，统计有多少比例有字幕。如果低于 60%，产品体验不可控。

---

### 探针 #3：部署门槛

**假设：** 用户能顺利安装并运行

**残酷真相：**
- weasyprint 需要 GTK/Cairo 系统依赖
- 目标用户是"B站重度用户"，但安装门槛把他们挡在门外
- Cookie 手动配置流程对非技术用户不友好

**改进：** 已实现扫码登录，降低 Cookie 配置门槛。下一步考虑纯 Markdown 输出模式。

---

### 探针 #4：体裁路由准确率

**假设：** LLM 能正确判断视频体裁

**残酷真相：**
- 没有任何准确率测试数据
- 体裁判断错误 → 用错提示词 → 总结结构不对 → 用户觉得"这总结不对味"
- 10 种体裁意味着 10 种出错可能，但没有 fallback 机制

**验证方式：** 取 20 个视频，手动标注体裁，和 LLM 判断对比，统计准确率。低于 80% 需要优化。

---

### 探针 #5：记忆系统膨胀

**假设：** 存储 summary 数据让工具更有价值

**残酷真相：**
- `processed.json` 会随时间膨胀——100 个视频的 summary 数据可能超过 1MB
- 全文搜索（search_by_keyword）是线性扫描，数据量大了会变慢
- 没有数据清理机制，用户无法删除不需要的记录

## 三、Positioning Statement：市场定位

### Geoffrey Moore 定位框架

**价值主张：**

> **For** B站重度用户，收藏夹中有大量未消化的干货视频
> **that need** 用最低时间成本把"收藏焦虑"转化为"已掌握的知识"
> **bilibili-reader is a** 视频内容智能提炼工具
> **that** 自动从收藏夹选取视频，读取字幕/评论/弹幕，生成中英双语结构化学习笔记

**差异化声明：**

> **Unlike** 手动看视频 / B站稍后再看 / YouTube 摘要插件
> **bilibili-reader provides** 基于视频体裁的定制化知识提炼——不是简介，而是可执行笔记

### 竞品差异化矩阵

| 维度 | bilibili-reader | 手动看视频 | YouTube 摘要插件 | B站稍后再看 |
|------|----------------|-----------|----------------|------------|
| 时间成本 | 低（自动生成） | 高（看完全部） | 中（需打开视频） | 低（但只是列表） |
| 知识结构化 | 高（体裁定制） | 无 | 低（通用摘要） | 无 |
| 覆盖平台 | B站 | 任意 | YouTube | B站 |
| 离线阅读 | PDF/Markdown | 不支持 | 不支持 | 不支持 |
| 知识积累 | 有记忆系统 | 无 | 无 | 无 |
| 语言 | 中英双语 | 原语言 | 英文为主 | 原语言 |

**核心洞察：你的真正竞争对手不是其他工具，而是"什么都不做"**——用户收藏了不看，也不会主动运行一个命令去生成总结。

## 四、社区反响预测

### GitHub：中等偏上

**有利因素：**
- 解决了一个真实痛点（收藏焦虑），容易引发共鸣
- 代码结构清晰，README 详尽，降低了上手门槛
- 中英双语总结是差异化亮点

**不利因素：**
- weasyprint 系统依赖会劝退一部分人
- 需要 B站 Cookie（涉及账号安全顾虑）
- 目标用户（B站重度用户）和 GitHub 用户重叠度有限

**预期：** 50-200 stars，取决于 README 质量和是否有中文社区推广

### Bilibili：高（如果宣传得当）

**有利因素：**
- B站用户对"收藏焦虑"有强烈共鸣
- "AI 帮你看视频"是 B站科技区的热门话题
- 可以用实际案例展示效果（选一个热门视频，展示总结 PDF）

**不利因素：**
- B站用户对"需要装 Python"的工具有天然抵触
- Cookie 安全问题会被评论区放大

**预期：** 如果做成视频教程，1-5 万播放，评论区会很活跃

### TikTok / YouTube：低

- 平台受众和目标用户不匹配
- "帮我总结 B站视频"对海外用户没有共鸣
- 除非做成"AI 工具测评"类内容，否则传播力有限

## 五、宣传策略

### GitHub README 优化

- 加一个 GIF 演示：展示从运行命令到生成 PDF 的完整流程
- 在最前面加一个"30 秒了解"区块：一张图说清楚这是什么
- 加"无 weasyprint 模式"：支持纯 Markdown 输出，降低安装门槛

### B站视频宣传（核心渠道）

**视频标题建议：**
- "我用 AI 帮自己看了 100 个 B站收藏视频，结果..."
- "收藏了不看？让 AI 帮你做视频笔记"
- "Python 自动总结 B站视频，生成双语 PDF"

**视频结构：**
1. 开头：展示收藏夹里几百个未看视频（引发共鸣），简单介绍一下
2. 中间：运行工具，展示生成过程和 PDF 效果
3. 结尾：给出 GitHub 链接，教观众安装

### 降低门槛的改进

- 支持纯 Markdown 输出（去掉 weasyprint 依赖）
- 提供在线版本（Streamlit/Gradio demo，用户不需要本地安装）
- 支持扫码登录（已实现，替代手动复制 Cookie）

## 六、迭代路线图

```
目标：让用户真正消化收藏内容
│
├─ Opportunity 1：降低"启动成本"

├─ Opportunity 3：从"单次消费"到"知识积累"
```

## 七、提示词优化总结

### 新增的 4 个通用学习字段（所有体裁）

| 字段 | 作用 |
|------|------|
| `prerequisites` | 前置知识——学习本内容需要什么基础 |
| `difficulty` | 难度评级——入门/进阶/高级 |
| `next_steps` | 后续行动——看完后应该做什么 |
| `key_misconceptions` | 常见误解——关于这个话题最常见的错误认知 |

### 各体裁新增的专用字段

| 体裁 | 新增字段 | 作用 |
|------|---------|------|
| 技术教程 | `expected_outcome` | 验证步骤：怎么知道做对了 |
| 学科教育 | `exam_format` | 考试出题形式 |
| 语言学习 | `vocabulary_list` | 结构化词汇表 |
| 深度解析 | `data_sources` | 引用的数据/证据来源 |
| 方法论 | `practice_template` | 填空式练习模板 |
| 职场 | `scripts_templates` | 可直接用的话术模板 |
| 创意 | `reference_works` | 参考作品/风格 |
| 书籍 | `key_quotes` | 关键引述 |
| 生活 | `materials_list` | 材料清单 |
| 通用 | `related_topics` | 相关话题推荐 |
